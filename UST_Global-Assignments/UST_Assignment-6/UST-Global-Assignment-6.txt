Scenario

You are building a small online bookstore database. The database should store information about authors, books, customers, and orders.

You need to create tables, insert data, perform CRUD operations, and write queries.

Part 1: Create Tables

Create the following tables with proper data types and constraints:

Authors Table
Column	Type	Constraint
author_id	INT	PRIMARY KEY, AUTO_INCREMENT
name		VARCHAR(100)	NOT NULL
country		VARCHAR(50)
	
Books Table
Column	Type	Constraint
book_id	INT	PRIMARY KEY, AUTO_INCREMENT
title	VARCHAR(100)	NOT NULL
author_id	INT	FOREIGN KEY REFERENCES Authors(author_id)
price	DECIMAL(10,2)	NOT NULL
stock	INT	DEFAULT 0

Customers Table
Column	Type	Constraint
customer_id	INT	PRIMARY KEY, AUTO_INCREMENT
name	VARCHAR(100)	NOT NULL
email	VARCHAR(100)	UNIQUE, NOT NULL

Orders Table
Column	Type	Constraint
order_id	INT	PRIMARY KEY, AUTO_INCREMENT
customer_id	INT	FOREIGN KEY REFERENCES Customers(customer_id)
book_id	INT	FOREIGN KEY REFERENCES Books(book_id)
quantity	INT	NOT NULL
order_date	DATE	NOT NULL

Part 2: Insert Sample Data

Insert at least 5 authors

Insert at least 10 books

Insert at least 3 customers

Insert 5 orders for different customers

Part 3: CRUD Operations

Insert a new author and a new book by that author.

Update the price of a book.

Delete a customer from the database.

Insert a new order for a customer.

Part 4: Queries

List all books along with their author names.

Find all orders made by a particular customer.

List books with stock less than 5.

Find the total number of books sold per book.

Find the total revenue generated from all orders.

Part 5: Advanced Queries

Find the top 3 bestselling books (highest quantity sold).

List authors who have more than 2 books in the database.

Display customers who never placed an order.

Show the total revenue generated by each author.

Optional Bonus

Create views for:

Books and their available stock.

Orders with customer names and book titles.
